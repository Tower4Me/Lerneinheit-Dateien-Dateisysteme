Modul 01 · Bits & Bytes

Bits & Bytes

Alles was ein Computer speichert, verarbeitet oder überträgt, besteht auf dem untersten Level aus zwei Zuständen: Spannung an oder aus. Das sind 1 und 0. Darauf baut alles auf.

Das Bit

Ein Bit (Binary Digit = binäre Stelle) ist die kleinste mögliche Informationseinheit. Es kann genau zwei Zustände annehmen: 1 oder 0. An oder aus. Wahr oder falsch.

Ein einzelnes Bit ist für sich wenig nützlich. Erst durch Kombination mehrerer Bits entstehen aussagekräftige Werte.

Bit

1 oder 0

Kleinste Einheit. Kommt von "binary digit". Ein Bit kann nur zwei verschiedene Werte haben.

Nibble

4 Bits

Halb so groß wie ein Byte. Kann Werte von 0 bis 15 darstellen. Entspricht genau einer Hex-Stelle.

Byte

8 Bits

Die grundlegende Speichereinheit. Kann 28 = 256 verschiedene Werte annehmen (0 bis 255).

Kilobyte (KB)

1.024 Byte

Im binären System: 210 = 1.024 Byte. Im Dezimalsystem (Festplattenhersteller): 1.000 Byte.

Warum 8 Bits pro Byte?

Das ist historisch gewachsen. Frühe Computer nutzten 6 oder 7 Bits. IBM setzte 1964 mit dem System/360 den Standard auf 8 Bits, weil das reicht um alle druckbaren Zeichen plus Steuerzeichen darzustellen und gut durch 2 teilbar ist.

Was kann ein Byte darstellen?

Mit 8 Bits sind genau 256 Kombinationen möglich (2 hoch 8). Das reicht für:

Beispiel01000101 = 0x65 = 'e' = Dezimal 101 0x bedeutet: diese Zahl ist hexadezimal (Basis 16) angegeben. 0x65 ist nicht "null x 65", sondern das Präfix für eine Hex-Zahl. 6 * 16 + 5 = 101 -- das ist derselbe Wert wie Dezimal 101.
0x erklärt

0x ist ein Präfix das anzeigt: "die folgende Zahl ist hexadezimal". 0x65 bedeutet also: Hex-Zahl 65. Das entspricht Dezimal 101. Genauso zeigt 0b eine Binärzahl an: 0b01000101 = dieselbe 101.

Binär vs. Dezimal – der Unterschied

Festplattenhersteller werben mit Dezimal-GB (1 GB = 1.000.000.000 Byte). Windows zeigt aber binäre GiB (Gibibyte: 1 GiB = 1.073.741.824 Byte). Eine "1 TB"-Festplatte zeigt Windows deshalb als ~931 GB an. Das ist kein Fehler, nur zwei verschiedene Definitionen.

Zahlensysteme

Computer rechnen binär, Menschen rechnen dezimal, und Entwickler lesen Hex. Alle drei Systeme beschreiben dieselben Werte – nur mit unterschiedlichen Symbolen.

Drei Systeme, ein Wert

Dezimal (Basis 10)

0–9

Das System das Menschen seit Jahrtausenden nutzen. 10 Symbole, weil wir 10 Finger haben. Stellenwert: 1, 10, 100, 1000...

Binär (Basis 2)

0 und 1

Das System der Computer. Nur 2 Symbole, weil Hardware nur zwei Zustände kennt. Stellenwert: 1, 2, 4, 8, 16...

Hexadezimal (Basis 16)

0–9 und A–F

16 Symbole. A=10, B=11, ... F=15. Sehr kompakt: 2 Hex-Stellen = 1 Byte. Deshalb überall in der IT.

Beispiel: Wert 255Dezimal: 255 Binär: 11111111 -- 8 Bits, alle auf 1 Hexadezimal: 0xFF -- F=15, also 15*16 + 15 = 255 Dezimal: 65 Binär: 01000001 -- 64+1 = 65 Hexadezimal: 0x41 -- 4*16 + 1 = 65 = 'A' in ASCII

Wie liest man Binär?

Jede Stelle hat einen Stellenwert der sich verdoppelt: von rechts nach links 1, 2, 4, 8, 16, 32, 64, 128. Du addierst die Werte der Stellen die eine 1 haben.

Beispiel: 0b10110010Stelle: 128 64 32 16 8 4 2 1 Bit: 1 0 1 1 0 0 1 0 Wert: 128 + 0 + 32 + 16 + 0 + 0 + 2 + 0 = 178

Wie liest man Hexadezimal?

Jede Hex-Stelle entspricht genau 4 Bits (einem Nibble). Das macht Hex so praktisch: ein Byte = 2 Hex-Stellen, immer.

Hex lesen0xB2 = B * 16 + 2 = 11 * 16 + 2 = 176 + 2 = 178 -- Schnelle Umrechnung: jede Hex-Stelle = 4 Bits B = 11 = 1011 2 = 2 = 0010 B2 = 10110010 = 0b10110010 = 178
